# RMR Website — Lessons

_Durable wins, patterns, and lessons learned during this project._

- No-gap background audio across static HTML pages requires same-document soft navigation. Keep shared scripts under `website/scripts/` idempotent so page markup can be swapped without restarting persistent UI state.
- Netlify app domain is `https://rollmyretirement.netlify.app`. Keep deployment/domain details in `docs/project/deployment.md` so agents do not have to infer the Netlify target from GitHub or local state.
- Persistent shared scripts that bind document-level interactions should use named handlers and refresh them in `ensureMounted()`; boolean "already attached" flags can drift after soft navigation and leave click/audio behavior stale.
- Mobile tactile audio should not wait for the synthetic `click` event. For touch/coarse-pointer cues, fire on validated pointer/touch release, suppress the follow-up click, and cancel when movement suggests scrolling.
- On mobile/iOS, avoid using an `HTMLAudio` sample as a fallback for tiny UI tap cues when background music is already playing. Use Web Audio for mobile click cues once primed, skip unprimed taps if necessary, and let branded logo taps prioritize the logo animation over the generic click sound.
- Before shipping new visual sections, check `DESIGN.md` typography and spacing scales instead of improvising one-off hero-sized type or decorative marks. For width-animated image panels, stabilize the media crop dimensions so panel expansion reveals the crop without perceived image zoom.
- Sponsor and advertising-section body copy must be checked against `DESIGN.md` and the styleguide HTML for both wording and type treatment. Do not promote explanatory sponsor copy into oversized lead styling unless the design system calls for a lead; keep it on the body-copy scale.
- New ideas must pass a cohesion check before shipping: compare the addition against nearby sections, `DESIGN.md`, and the local styleguide for type scale, spacing rhythm, visual weight, motion behavior, and tone. A section can be individually polished and still be wrong if it does not belong to the whole page.
- When matching an existing GSAP entrance animation, remove competing CSS `transform`/`opacity` transitions from the animated target element. Keep hover/layout transitions on inner surfaces or unrelated properties so the copied motion actually feels identical.
- For ScrollTrigger reveals that must start invisible, explicitly set the hidden state with `gsap.set()` before creating the timeline. Do not rely on a deferred `fromTo` start state when the expected visual behavior is true `opacity: 0` before the section enters.
- When a one-time ScrollTrigger section can be opened by URL hash, verify that hash path separately. If the browser or smooth-scroll layer does not land on the anchor before ScrollTrigger evaluates, add a scoped fallback that scrolls to the anchor and plays the same timeline when the section enters its trigger zone.
- Any pre-first-paint document flag used for a hard-load visual state must have an explicit soft-navigation cleanup path in `router.js`. Otherwise a flag set on one document can leak into the next body swap and create a false entry state.
- Autonomous loops such as hero text swaps must start after the entrance timeline completes, not on page-script load. If a loader or curtain delays the hero, a global `setTimeout` can fire while the entry choreography is still running and make the section look broken.
- Fake loaders for this brand should feel calm and designed, not pseudo-technical. Prefer a predictable eased 00-100 progress path with a short end hold over staged "realistic" jumps when the visual direction is quiet and editorial.
