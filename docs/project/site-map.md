# Site Map

Production uses clean Netlify routes. Source pages live under `website/pages/`; root `rmr-*.html` files are generated compatibility outputs.

| Route | Source | Purpose |
| --- | --- | --- |
| `/` | `website/pages/home.html` | Consumer homepage |
| `/advisors` | `website/pages/advisors.html` | Advisor network landing page |
| `/contact` | `website/pages/contact.html` | Consumer contact/get-started form |
| `/insights` | `website/pages/insights.html` | Blog index |
| `/insights/:slug` | `website/pages/article.html` | Blog post detail renderer |

## Navigation

- Homepage nav anchors: About, Process, Network, FAQs.
- Cross-page links: Insights, For Advisors, Let's Talk.
- Advisor page anchors: Why RMR, How It Works, Standards, FAQ, Apply.
- Blog and contact pages use compact cross-page nav.
- Footer links are generated by `website/scripts/footer.js` according to the active page.

## Legacy Compatibility

`netlify.toml` redirects generated `.html` paths to clean production routes. Keep those redirects unless the generated output names are intentionally changed.
